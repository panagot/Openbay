# Ask Grok AI: Propose 3 grant milestones for peaq

---

## Grant structure

- **Total ask:** $15,000
- **Split:** $5,000 per milestone (M1, M2, M3)
- **FTE:** 1 (solo or small team)
- **Duration per milestone:** ~4–6 weeks (you can suggest exact timelines)

The form requires for each milestone:
1. **Estimate:** How long to complete, FTE, grant amount required.
2. **Deliverables:** List including licenses (if any), documentation, tests, contracts/modules, dApps, etc.

---

## Our project: Virtual DeCharge World

- **What it is:** Operating system for independent EV charger owners on peaq. Turn parking spots/garages into revenue-generating charging nodes via machine identity (peaq ID), verifiable sessions (peaq verify), and P2P payments (peaq pay).
- **Current state:** Working prototype already built: grid (mint, deploy, register on peaq), map (Google Maps + demo stations), wallet (MetaMask + local dev), session simulation, POINTS (in-app only, no ERC-20). Live demo: https://powering-the-e-volution-on-peaq.vercel.app/
- **What we do NOT want in the milestones:** We are **not** committing to open-sourcing the repo or making GitHub public. We are **not** including an ERC-20 token; POINTS stay in-app only.

---

## What we want each milestone to focus on

- **Milestone 1:** Launch and “normal procedures” — acquire **domain name**, set up **production website**, create **social media** presence (X, Discord, etc.), polish the existing dApp for production, documentation, basic testing. The demo is already built; M1 should be straightforward (website, domain, socials, go-live).
- **Milestone 2:** Trust layer and real data — **peaq verify** integration, **Trust Score** in the UI, at least one **real data source** (EV charger API or IoT/OCPP prototype), draft **public API/SDK** for charger discovery, documentation. Still on testnet/staging.
- **Milestone 3:** **Mainnet** deployment so **real users** can interact. Integrate **peaq pay** for charging sessions. **Pilot** outreach and **promotion** (e.g. garage owners, EV communities, socials). Final documentation.
